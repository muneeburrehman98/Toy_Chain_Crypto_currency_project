ToyChain Blockchain: Final Implementation Report

1. Project Overview

ToyChain is a functional blockchain simulation designed to demonstrate the core principles of cryptography, data integrity, and distributed ledgers. The project evolved from a command-line utility into a modern GUI application with real-time visualization.

2. Technical Architecture & Analysis

The system is built as a modular C++ application with a clean separation between core logic and the presentation layer.

Core Components

  • Blockchain: The central manager that maintains the chain of blocks and the transaction pool.
  • Block: A container for transactions, metadata (index, timestamp, nonce), and linking hashes.
  • Merkle Tree: A cryptographic tool used to summarize all transactions in a block into a single "Merkle Root" for efficient integrity checks.
  • GUI Dashboard: A modern wxWidgets interface providing interactive control and real-time status updates.

3. Data Structures Used

Component Data Structure Rationale
Blockchain std::vector<Block> Blocks are stored sequentially. A vector provides O(1) access to any block and efficient appending.
Mempool std::priority_queue Transactions are sorted by "Fee" so that miners pick the most profitable ones first.
Transaction List std::vector<Transaction> Used within each block to store a fixed set of transactions in a contiguous memory block.
Merkle Tree Recursive Binary Tree (implicit) Implemented using a recursive std::vector approach to compute the root hash.

4. Cryptographic Implementation

  • Hashing: Powered by the picosha2 library (SHA-256). Every block contains the hash of the previous block, creating an immutable chain.
  • Proof of Work: Implements a difficulty system where a block hash must start with a specific number of zeros.
  • Tamper Detection: The system re-calculates hashes and Merkle Roots during validation to detect unauthorized changes.
